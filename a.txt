这是 RainJava 的中文介绍：

---

RainJava

在 Minecraft 里写 Java 代码。即时编译，无需重启。

RainJava 是一款强大的游戏内脚本模组，将完整的 Java 语言带入 Minecraft。在游戏运行期间编写、编译和执行 Java 代码，修改游戏行为、注入方法、转换字节码，并且能够立即看到变化。

---

你能做什么

实时编程

创建在启动时、服务器启动时或客户端初始化时执行的 Java 脚本。使用任意文本编辑器编辑代码，保存文件后输入 `/java reload` 即可即时应用更改。内置的 Eclipse 编译器自动处理一切，即使你的系统没有安装 Java 编译器也能正常工作。

Mixin 注入

直接以源代码形式编写 Mixin 类，向任意 Minecraft 方法注入代码。使用 `@Inject.At("HEAD")` 定位方法开头，用 `"RETURN"` 捕获返回点，或用 `"TAIL"` 拦截最终退出位置。所有游戏类都可修改。

CoreMod 与 ASM 转换

对于深度修改，编写 CoreMod 插件，在 JVM 加载之前转换原始字节码。访问任意已加载类的每条指令、字段和方法。这与传统 Forge 核心模组拥有同等级别的权限，但通过简单的 Java 源文件实现，按需编译。

资源包与数据包

直接在 RainJava 项目中捆绑自定义纹理、模型、音效、配方、战利品表和标签。`assets/` 和 `data/` 文件夹的工作方式与标准资源包和数据包完全相同，随脚本自动加载。

---

工作原理

将 Java 源文件放入 Minecraft 目录内的 `RainJava/` 文件夹。文件夹结构决定代码执行时机：

- `startup/` — 游戏初始化时执行一次
- `server/` — 每次服务器启动时执行
- `client/` — 客户端初始化时执行
- `mixins/` — 用于运行时方法注入的 Mixin 类
- `coremod/` — CoreMod 插件和 ASM 转换器

每个脚本声明一个简单的 `init()` 方法。框架自动发现、编译和执行。错误信息会写入专用日志文件，并可通过 `/java errors` 命令在游戏内聊天中查看。

---

必备文档

每次安装都会在 `RainJava/` 文件夹内包含一份详尽的 `README.txt` 文件。该文件包含包声明、Mixin 语法、注入点、CoreMod 架构、可用命令和故障排查的完整参考材料，所有内容均可离线查阅，无需网络连接。

README 随每次更新同步更新，以反映当前 API 行为和支持的功能。遇到语法问题、错误信息或高级用法疑问时，请优先查阅此文件。

---

命令

- `/java reload [startup|server|client]` — 热重载脚本，无需重启
- `/java hand getId` — 显示手持物品的注册 ID
- `/java hand getClass` — 显示手持物品的 Java 类名
- `/java errors [类型]` — 在聊天中显示编译和运行时错误

---

运行要求

- Minecraft 1.20.1 + Forge
- Java 17 或更高版本
- 与标准 Forge 模组和 Mixin 系统兼容
